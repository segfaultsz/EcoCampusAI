<objective>
Fix console error: `Could not load models: Cannot read properties of undefined (reading 'oak1-lod3')`.

This crash originates from Mapbox GL JS v3.20's **Standard style** (`mapbox://styles/mapbox/standard`).
The Standard style includes built-in 3D vegetation models (trees like `oak1-lod3`). When the Mapbox access
token is missing/invalid/undefined, the style partially loads but its internal 3D model manifest returns
`undefined`, causing a crash when the renderer tries to access model properties like `oak1-lod3`.
</objective>

<context>
## Root Cause Analysis

**File:** `frontend/src/app/campus/page.js` (line 133)

**Data flow:**
1. `page.js` dynamically imports `mapbox-gl` (v3.20.0)
2. Sets `mapboxgl.accessToken = process.env.NEXT_PUBLIC_MAPBOX_TOKEN`
3. Creates a map with `style: 'mapbox://styles/mapbox/standard'` and `pitch: 55` (3D tilt)
4. Standard style internally loads 3D models for landmarks, trees, buildings — including `oak1-lod3` (an oak tree LOD mesh)
5. **BUG**: `NEXT_PUBLIC_MAPBOX_TOKEN` is never defined — there is no `.env.local` file in `frontend/`, and the token is not in `backend/.env`. So `process.env.NEXT_PUBLIC_MAPBOX_TOKEN` evaluates to `undefined`.
6. With `accessToken = undefined`, Mapbox authenticates with an empty token. The style loads partially, but the 3D model manifest endpoint returns an error/empty response. The renderer then crashes trying to read `undefined['oak1-lod3']`.

**Why `oak1-lod3` specifically:**
Mapbox Standard uses a "model layer" for 3D vegetation. `oak1-lod3` is an LOD (Level of Detail) variant of an oak tree model. The model batch data is fetched from `https://api.mapbox.com/models/v1/mapbox/...` which requires a valid public token. When auth fails, the response is undefined, causing the property access crash.

**Evidence:**
- `frontend/package.json`: `"mapbox-gl": "^3.20.0"` (Standard style with 3D models)
- `campus/page.js:138`: `style: 'mapbox://styles/mapbox/standard'` (uses Standard, not streets-v12)
- `campus/page.js:133`: `mapboxgl.accessToken = process.env.NEXT_PUBLIC_MAPBOX_TOKEN` (reads undefined env var)
- `frontend/.env.local`: **DOES NOT EXIST** (confirmed via file search)
- No fallback or error handling on map creation
</context>
